# چرا `lms.psychoanalysis.ir` → 404 OpenResty و چطور درست شود

## علت

- روی ماشین آنیستیتو (مثلاً `tpi-ims`) اینترفیس واقعی **`192.168.91.9`** است؛ آدرس عمومی **`80.191.11.129`** روی لبهٔ شبکه (روتر/Proxmox/هاست) است.
- از **داخل** همان VM با `curl` به `127.0.0.1:80` پاسخ **Apache** می‌گیرید (ریدایرکت به `/anistito/`).
- از **اینترنت** با اتصال به **`80.191.11.129:80`** پاسخ **OpenResty** و **404** می‌گیرید.

یعنی **پورت ۸۰ روی IP عمومی به این VM فوروارد نشده**؛ ترافیک HTTP به سرویس دیگری (OpenResty روی همان دیتاسنتر یا IP دیگر) می‌رسد.

تا وقتی این مسیر اصلاح نشود، **دامنهٔ lms حتی با Apache درست روی VM از بیرون درست دیده نمی‌شود.**

## راه‌حل‌ها (یکی را انتخاب کنید)

### الف) فوروارد پورت روی لبهٔ شبکه (ساده برای همان IP که الان `lms` به آن اشاره می‌کند)

روی **روتر / Proxmox / فایروال** که **`80.191.11.129`** را نگه می‌دارد:

- **TCP 80** و در صورت نیاز **443** را به **`192.168.91.9:80`** (و 443) همان VM آنیستیتو **DNAT** کنید.

بعد از آن، `http://lms.psychoanalysis.ir/` باید همان رفتار `curl` با `Host` روی خود VM را از بیرون نشان دهد.

### ب) اگر ترافیک وب همه از یک OpenResty مرکزی می‌گذرد (مثل سایت اصلی)

- رکورد DNS **`lms.psychoanalysis.ir`** را مثل سایت اصلی به همان IPای بگذارید که **OpenResty** روی آن گوش می‌دهد (مثلاً همان **`80.191.11.134`** که برای `psychoanalysis.ir` استفاده می‌شود، اگر معماری شما این است).
- در **OpenResty/nginx** یک `server` با `server_name lms.psychoanalysis.ir` اضافه کنید و به بک‌اند داخلی پروکسی کنید، مثلاً:

```nginx
server {
    listen 80;
    server_name lms.psychoanalysis.ir;

    location / {
        proxy_pass http://192.168.91.9:80;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

آدرس `192.168.91.9` را با IP واقعی VM آنیستیتو در شبکهٔ داخلی خودتان جایگزین کنید؛ اگر OpenResty روی همان میزبان نیست، باید مسیریابی L3/VPN بین سرورها باز باشد.

### ج) HTTPS

بعد از درست شدن HTTP، روی **همان لایه‌ای که ترافیک را می‌گیرید** (Apache روی VM یا OpenResty) گواهی **Let’s Encrypt** برای `lms.psychoanalysis.ir` بگیرید تا با سایت اصلی هم‌تراز شود.

## جمع‌بندی

404 OpenResty یعنی **درخواست به وب‌سرور آنیستیتو نمی‌رسد**؛ مشکل **DNS به تنهایی نیست**، بلکه **مسیر پورت ۸۰/۴۴۳ از اینترنت تا Apache داخل VM** است که باید روی زیرساخت تنظیم شود.
