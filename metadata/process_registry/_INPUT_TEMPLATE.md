# قالب ورود فرایند جدید

وقتی فرایند جدید می‌فرستید، این ساختار را رعایت کنید تا تبدیل دقیق‌تر باشد.

---

## ۱. نام و شناسه
- **نام فارسی:** 
- **کد (انگلیسی، با آندرلاین):** مثلاً `my_new_process`

## ۲. فلسفه و هدف
(چرا این فرایند وجود دارد؟)

## ۳. گام‌های اجرایی (دستورالعمل)
برای هر گام:
- چه کسی اقدام می‌کند؟ (دانشجو، کمیته، سیستم، …)
- چه کاری انجام می‌شود؟
- شرط یا تصمیم‌گیری وجود دارد؟
- SLA یا مهلت مشخص است؟
- چه اعلان/اکشنی باید انجام شود؟

## ۴. فلوچارت (در صورت وجود)
توضیح یا تصویر فلوچارت را ضمیمه کنید.

---

## مثال پر شده
```
نام فارسی: مرخصی آموزشی موقت
کد: educational_leave

فلسفه: مدیریت وقفه تحصیلی با حفظ الزامات آموزش.

گام ۱: دانشجو درخواست می‌دهد (۱ یا ۲ ترم). اگر انترن و ۲ ترم → هشدار.
گام ۲: کمیته ظرف ۷ روز جلسه تعیین کند. اگر نکرد → هشدار به معاون.
...
```
