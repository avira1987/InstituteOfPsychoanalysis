# ثبات فرایندها — ورود متن انسانی و تبدیل به JSON

## هدف
ساختار واحدی برای ثبت فرایندهای ارسالی، تبدیل آن‌ها به JSON و پیگیری نواقص و وابستگی‌ها.

---

## ساختار پوشه‌ها

```
process_registry/
├── README.md              ← همین فایل
├── INDEX.json             ← فهرست ماشین‌خوان فرایندها و وضعیت آن‌ها
├── GAPS.json              ← نواقص سیستم و وابستگی‌های حل‌نشده
├── _INPUT_TEMPLATE.md      ← قالب ورود فرایند جدید
├── _AI_INSTRUCTIONS.md     ← دستورالعمل برای چت هوشمند
└── processes/
    └── {process_code}/
        ├── 01_input.md     ← متن و توضیحات ورودی (انسان)
        ├── 02_flowchart.md ← یادداشت فلوچارت (اختیاری)
        ├── 03_output.json  ← JSON تولیدشده (یا ارجاع به metadata/processes/)
        └── 04_status.md    ← وضعیت، تاریخ‌ها، نواقص، وابستگی‌ها
```

---

## نحوه استفاده

### برای شما (انسان)
1. فرایند جدید را طبق `_INPUT_TEMPLATE.md` بنویسید.
2. آن را در `processes/{کد_فرایند}/01_input.md` ذخیره کنید.
3. فلوچارت یا یادداشت‌ها را در `02_flowchart.md` قرار دهید.
4. به چت هوشمند بگویید: «این فرایند را بخوان و JSON بساز.»
5. چت `INDEX.json` و `GAPS.json` را به‌روزرسانی می‌کند.

### برای چت هوشمند
1. قبل از هر کار، `INDEX.json` و `GAPS.json` را بخوان.
2. متن ورودی را از `01_input.md` بخوان.
3. JSON تولید و در `03_output.json` و `metadata/processes/{code}.json` ذخیره کن.
4. `04_status.md` را به‌روزرسانی کن.
5. `INDEX.json` و در صورت نیاز `GAPS.json` را به‌روزرسانی کن.
